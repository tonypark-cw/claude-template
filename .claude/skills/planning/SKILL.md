# Planner — 구조화된 요구사항 수집 + PRD 생성

핑퐁 Q&A로 요구사항을 명확히 한 뒤 PRD를 생성하고 `docs/prd.md`에 저장.

## 핵심 규칙

- **기획 중 코드 생성 절대 금지**
- **모든 질문이 끝나기 전 PRD 초안 작성 금지**
- 답변이 모호하면 다음 질문으로 넘어가기 전 **후속 질문 1개** 추가

## 프로세스

### Step 1: 핑퐁 Q&A — 한 번에 질문 하나

아래 순서대로 질문을 하나씩 던지고 답을 받은 뒤 다음 질문으로 진행.

```
Q1. 이 제품을 쓸 사람은 누구이고, 그들이 지금 겪는 핵심 불편함은 무엇인가요?

Q2. MVP가 반드시 해야 하는 딱 한 가지 기능은 무엇인가요?

Q3. 우선순위 순으로 핵심 기능 3~5개를 꼽아 주세요.

Q4. 성공을 어떻게 측정할 건가요? (구체적인 수치나 기준)

Q5. 이번 범위에서 명시적으로 제외할 것은 무엇인가요?

Q6. 기술 제약이 있나요? (언어, 프레임워크, 마감일 등)
```

답변이 모호하면 후속 질문 1개:
```
예: "'빠르게'라고 하셨는데, 응답 시간 기준이 있나요? 예를 들어 3초 이내?"
```

### Step 2: PRD 생성

`templates/prd-template.md`의 7개 섹션 구조를 따라 작성.
수집한 답변으로 플레이스홀더를 채움.

### Step 3: docs/prd.md 저장

```bash
# docs/ 디렉토리가 없으면 생성
mkdir -p docs
```

생성한 PRD를 `docs/prd.md`에 저장. 이 파일이 SSOT.

### Step 4: 다음 단계 안내

```
PRD가 docs/prd.md에 저장되었습니다.

다음 단계:
- 구현 시작: "docs/prd.md를 읽고 Phase 1을 구현해줘"
- 평가 실행: /evaluate
- 검토 요청: /review
```

## 원칙

1. **질문 순서 고정** — 순서를 바꾸거나 건너뛰지 않음
2. **답변 대기** — 사용자가 답하기 전에 다음 질문 금지
3. **추측 금지** — 답변에서 명확히 확인한 것만 PRD에 기재
4. **범위 고정** — PRD 확정 후 추가 요구사항은 다음 버전으로
